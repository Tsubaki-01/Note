### Prompt 压缩是怎么实现的？

目前业界有很多种压缩方法，从简单粗暴到高度硬核都有：

1. **词汇过滤与剔除（Lexical Filtering）** 这是最基础的方法。通过算法直接删掉文本中不影响理解的停用词（比如英文里的 "the"、"a"，中文里的“的”、“地”）、多余的标点符号、空格或格式符。
2. **总结摘要法（Summarization）** 利用一个运行成本更低、速度更快的“小模型”（比如 GPT-3.5 或开源的 Llama），先把你那篇长长的背景文章做个摘要，然后再把这段精简后的摘要作为 Prompt 发给“大模型”（如 GPT-4）去执行最终任务。
3. **基于信息熵的智能压缩（Information Entropy）** 这是一种更高级的算法（例如微软提出的 *LLMLingua*）。它会去计算一句话里哪些词汇包含的信息量最高。如果去掉某些词，大模型依然能“猜”出这句话的意思，系统就会把这些多余的词无情删掉。压缩后的文本人类读起来可能会觉得语病百出、断断续续，但 AI 却能完美理解。
4. **软提示压缩（Soft Prompt / Embedding Compression）** 这种方法比较偏底层开发。它不再压缩“人类可读的文字”，而是把很长的文本直接转化成模型能理解的浓缩数学向量（Embeddings / Soft Prompts），相当于把文字变成了压缩包直接塞进 AI 的大脑。
