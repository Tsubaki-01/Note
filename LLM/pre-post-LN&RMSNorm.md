# post-LN

$$x_{l+1} = \text{LayerNorm}(x_l + \text{Sublayer}(x_l))$$

### 1. 梯度消失与梯度爆炸问题 (Vanishing/Exploding Gradients)

这是 Post-LN 最核心的问题。在 Post-LN 中，层归一化（LayerNorm）位于残差连接（Residual Connection）之后。

- **链式法则下的影响**：当你通过多个 Post-LN 层进行反向传播时，每一层的梯度都会受到 LayerNorm 权重的影响。由于每一层都在改变输出的尺度，靠近输入层的梯度（底层的梯度）往往会变得非常大或非常小。
- **数学直观**：在 Post-LN 中，由于残差分支的输出先相加再归一化，随着层数 $L$ 的增加，输出的期望方差会不断累积，导致 LayerNorm 的缩放因子变得极端，从而使得反向传播时的梯度流不稳定。

### 2. 对超参数极其敏感（尤其是学习率 Warm-up）

Post-LN 模型在训练初期非常脆弱，如果没有经过精心设计的**学习率预热 (Warm-up)**，模型几乎无法收敛。

- **初始状态不稳**：在训练刚开始时，权重是随机初始化的。Post-LN 的结构导致深层输出的梯度在初始化阶段就可能非常剧烈。
- **依赖 Warm-up**：为了防止模型在第一步训练时就“跑飞”（梯度爆炸导致权重被破坏），开发者必须使用非常小的学习率进行数千步的预热。相比之下，Pre-LN 往往可以在没有或只有极短预热的情况下稳定训练。

### 3. 深层模型训练困难 (Deep Model Instability)

随着模型层数的增加，Post-LN 的不稳定性呈指数级增长。

- **层数限制**：在不使用特殊初始化技巧（如 Fixup 或 ReZero）的情况下，很难训练超过 20-30 层的纯 Post-LN Transformer。
- **Pre-LN 的崛起**：这就是为什么现在的 GPT-3、Llama 等大模型几乎全部转向 **Pre-LN**（$\text{LayerNorm}$ 放在子层之前）的原因。Pre-LN 允许梯度更直接地通过残差通道流动，使得训练数百层的深层网络变得简单。

### 4. 恒等路径 (Identity Path) 的破坏

残差连接的核心思想是提供一条“恒等映射”通道，让信息可以无损地流向深层。

- **归一化的干扰**：在 Post-LN 中，每一层的输出在进入下一层之前都**必须**经过一次 LayerNorm 的重新缩放和偏移。这意味着原始的输入信号在传递过程中被不断地“扭曲”和重新分配。
- **信号衰减**：这种结构削弱了残差连接在保持梯度流顺畅方面的作用，使得深层神经元难以学习到对浅层特征的微调。

# pre-LN

$$x_{l+1} = x_l + \text{Sublayer}(\text{LayerNorm}(x_l))$$​

#### A. 保护“恒等映射”路径 (The Clean Residual Path)

残差连接（Residual Connection）的初衷是提供一条信息直达深层的“高速公路”。

- 在 **Pre-LN** 中，主干路径（$x_l$）是**线性叠加**的。这意味着梯度可以在反向传播时直接穿过所有残差层，而不受归一化层权重的剧烈干扰。
- 这种结构使得模型在初始化时，其表现更像是一个多个浅层网络的集成，极大降低了优化难度。

#### B. 解决梯度消失与爆炸 (Gradient Stability)

- **初始梯度稳定**：理论研究证明，在模型初始化阶段，Pre-LN 的梯度范数几乎不随层数增加而变化。
- **无需（或缩短）Warm-up**：Post-LN 对学习率预热（Warm-up）极度依赖，稍微设置不当模型就会崩掉。而 Pre-LN 非常稳健，很多情况下甚至可以不需要 Warm-up 或仅需极短的步数就能平稳启动训练。

#### C. 支持更深的网络架构 (Scalability)

这是大模型时代最重要的优点。

- 使用 Post-LN 训练超过 20 层的模型通常非常困难，容易在训练中途出现 Loss 突变（Spike）。
- **Pre-LN** 允许研究者轻松训练出数百层（如 175B 的 GPT-3 或 1000 层的实验模型）而不会出现数值不稳定的问题。

#### D. 超参数鲁棒性

Pre-LN 对学习率（Learning Rate）的波动不那么敏感。在超大规模分布式训练中，由于涉及数千张显卡，训练稳定性（Training Stability）的价值远高于微小的性能提升。一旦训练中途崩溃，重新启动的成本（计算资源、时间）是不可接受的。

### 为什么 LN 必须在 Sublayer 之前？

你可能会问：为什么不放在 Sublayer 之后但在残差加法之前？（即 $x + \text{LN}(F(x))$）

- **保护输入分布**：Sublayer（如 Self-Attention 或 FFN）对输入的分布非常敏感。Pre-LN 确保了每一层的子层在处理数据前，其输入都是经过标准化的（均值 0，方差 1）。这让子层的参数从训练第一步起就能在稳定的数值范围内工作。
- **控制方差爆炸**：由于 Pre-LN 的主干路径是不断累加的，如果没有这层 LN，子层 $F(x)$ 的输出方差可能会随着层数加深而迅速膨胀。在进入 $F(x)$ 之前先做 LN，本质上是限制了每个分支对主干路径的“干扰”强度。

# RMSNorm

标准 LayerNorm:



$$\bar{x}_i = \frac{x_i - \mu}{\sqrt{\frac{1}{n}\sum_{i=1}^n (x_i - \mu)^2 + \epsilon}} \cdot \gamma_i + \beta_i$$

RMSNorm:



$$\bar{x}_i = \frac{x_i}{\sqrt{\frac{1}{n}\sum_{i=1}^n x_i^2 + \epsilon}} \cdot \gamma_i$$

其中 $\text{RMS}(x) = \sqrt{\frac{1}{n}\sum_{i=1}^n x_i^2 + \epsilon}$。可以看到，它不再需要计算均值 $\mu$，也不再有偏置项 $\beta$。

#### A. 更高的计算效率

这是 RMSNorm 最直观的优点。

- **计算开销小**：省去了均值的计算以及后续的所有减法操作。在 GPU 上，这种简化能显著减少内存读写和计算延迟（大约能提升 10%~40% 的归一化层计算速度）。
- **参数量更少**：去掉了 $\beta$（偏置项）参数，虽然在超大模型中这部分节省的参数量微乎其微，但模型结构变得更加简洁。

#### B. 训练稳定性与收敛速度

- **保持向量方向**：RMSNorm 实际上是对神经元激活值的向量做了“投影”到固定长度球面的操作。它只关注激活值的**模长（Magnitude）**，而不改变向量的**方向**。
- **数值稳定性**：在混合精度训练（Mixed Precision Training）中，减少减法操作有助于降低由于浮点数精度限制带来的数值不稳定性（如 Round-off errors）。
- 无均值计算带来的精度误差，分母 RMS (x) 恒正，避免方差趋近于零导致的除零风险与梯度震荡。
- LayerNorm 的 “减均值” 是基于**当前层所有 token 的均值**做的，一旦均值偏移，相当于给每个 token 都减去了一个 “不稳定的数”，直接导致：
  - 归一化后的数值分布跟着均值一起漂移（比如均值从 0.1 变 0.5，所有 token 的归一化值都整体下降 0.4）；
  - 这种漂移会沿着网络层传递，越到深层越明显，最终引发严重的内部协变量偏移 —— 模型训练时要不断适配这种漂移，表现为 loss 震荡、收敛慢，甚至在万级 token 训练时直接不收敛。

#### C. 实证性能几乎等同于 LayerNorm

多项实验（包括 Llama 论文）证明，在各种下游任务上，RMSNorm 的表现与标准 LayerNorm 基本持平。既然性能一样，但速度更快、逻辑更简单，开发者自然倾向于选择 RMSNorm。
