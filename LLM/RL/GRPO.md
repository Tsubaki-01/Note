# GRPO：面向推理的组相对策略优化

## 一、为什么需要 GRPO？

PPO 为了计算优势函数，必须维护一个与 Actor 同等规模的 **Critic 模型**，导致训练时同时占用 4 个模型的显存（策略 + 参考 + 奖励模型 + Critic），成本极高。

DeepSeek 的洞察是：对于数学、代码等**可验证任务**，奖励的对错可以用规则直接判断，根本不需要神经网络来估值。既然 Critic 的唯一作用是提供一个"平均基线"，为何不用统计学直接替代它？

------

## 二、核心机制：用组内竞争替代价值网络

GRPO 的名字"Group Relative"道出了它的本质——**让同一个 prompt 的多个回复互相竞争，组内均值就是天然基线**。

对同一道题写 G 份答案，算出平均分，谁超过平均就强化，谁垫底就抑制——这就是 GRPO 的全部秘密。优势估计公式极为简洁：

$$\hat{A}_i = \frac{r_i - \text{mean}(r_1, \ldots, r_G)}{\text{std}(r_1, \ldots, r_G)}$$

类比考试的 z-score：不看绝对分，只看"在这批同学中排第几"。标准差归一化让不同难度的题目产生量级相当的梯度信号。

### 训练流程（Step-by-Step）

**步骤 1 — 组采样**：对每个 prompt $q$，从旧策略 $\pi_{\theta_{old}}$ 采样 **$G$ 个完整回复** ${o_1, o_2, \ldots, o_G}$（DeepSeekMath 中 $G=64$，DeepSeek-R1 中 $G=16$）。

**步骤 2 — 规则打分**：用奖励函数对每个回复打分，得到 ${r_1, r_2, \ldots, r_G}$。对于数学/代码题，直接用 Python 脚本验证答案对错，无需神经网络 RM。

**步骤 3 — 组内归一化**：计算每个回复的优势 $\hat{A}_i = (r_i - \mu) / \sigma$，均值充当基线，标准差保证梯度量级一致。

**步骤 4 — 优势分配**：在 outcome supervision 模式下（R1 标准设置），同一回复内所有 token 共享相同的优势值 $\hat{A}_{i,t} = \hat{A}_i$。

**步骤 5 — PPO-style 更新**：代入目标函数，小步更新 Actor 参数，清空数据进入下一个 prompt。

------

## 三、目标函数

$$J_{GRPO}(\theta) = \mathbb{E}_{x \sim P(x), \{y_i\}_{i=1}^G \sim \pi_{\theta_{old}}(y|x)} \left[ \frac{1}{G} \sum_{i=1}^G \left( \min \left( \frac{\pi_\theta(y_i|x)}{\pi_{\theta_{old}}(y_i|x)} \hat{A}_i, \;\text{clip}\left(\frac{\pi_\theta}{\pi_{\theta_{old}}}, 1-\epsilon, 1+\epsilon\right) \hat{A}_i \right) - \beta \mathbb{D}_{KL}(\pi_\theta \| \pi_{ref}) \right) \right]$$

与 PPO 相比有三处关键差异：

1. **优势估计**用组归一化代替 GAE + Critic
2. **KL 惩罚**作为显式项直接写入目标函数（而非嵌入逐 token 奖励）
3. 目标函数显式对 $G$​ 个样本和序列长度**取平均**

> GRPO 中的 per-token KL 采用近似形式：$D_{KL} = \frac{\pi_{ref}}{\pi_\theta} - \log\frac{\pi_{ref}}{\pi_\theta} - 1$，这是一种数值稳定的 KL 近似。

## 四、架构对比：只需 2 个模型

| 角色                     | PPO                 | GRPO                 |
| ------------------------ | ------------------- | -------------------- |
| Actor（策略模型）        | ✅                   | ✅                    |
| Reference（参考模型）    | ✅                   | ✅                    |
| Critic（价值模型）       | ✅ 与 Actor 同等规模 | ❌ 完全移除           |
| Reward Model（奖励模型） | ✅ 神经网络          | ❌ 规则脚本替代       |
| **需反向传播**           | **2 个模型**        | **1 个模型**         |
| **显存需求**             | **约 4× 模型大小**  | **约 1-2× 模型大小** |

------

## 五、GRPO 训练循环（Step-by-Step）

1. **采样 (Rollout)**：给出一个数学题 Prompt，让当前的 Actor 并行生成 $G=8$ 个推理过程和答案。
2. **规则打分 (Rule-based Reward)**：用代码脚本检查这 8 个答案。比如有 3 个算对了（得 1 分），5 个算错了（得 0 分）。
3. **群组标准化算优势**：把这 8 个分数 $[1, 1, 1, 0, 0, 0, 0, 0]$计算均值和标准差，得到这 8 个回答的群体相对优势 $\hat{A}_1 \dots \hat{A}_8$。（算对的优势为正，算错的优势为负）。
4. **计算 KL 惩罚**：将这 8 个回答输入 Reference 模型，算出概率，与 Actor 对比，得到 KL 惩罚值。
5. **PPO-Style 更新**：代入 GRPO 的公式，进行多次 Epoch 的反向传播，小步更新 Actor 的参数。
6. **清空数据，进入下一题。**

## 六、DeepSeek-R1 中的应用

### R1-Zero：最令人震撼的实验

完全不用 SFT 热启动，仅靠两种规则奖励从 base 模型纯 RL 训练：

- **准确性奖励**：答案是否正确（二值判断）
- **格式奖励**：是否使用 `<think>...</think>` 和 `<answer>...</answer>` 标签

结果：AIME 2024 从 **15.6% 提升至 71.0%**（pass@1），多数投票达 86.7%，匹配 OpenAI o1-0912。

更关键的是模型**自发涌现**的能力：长链推理、自我验证、纠错回溯——这些能力从未在训练数据中出现过。最著名的"顿悟时刻"（Aha moment）是模型在推理中途说出 *"Wait, wait. Wait."* 然后重新审视并修正了之前的推理。

### R1 完整四阶段流程

**阶段 1 — 冷启动 SFT**：用 R1-Zero 高质量输出微调 base 模型，避免纯 RL 早期的训练不稳定。

**阶段 2 — 推理导向 RL**：在数学/代码/逻辑任务上用 GRPO + 规则奖励（$\varepsilon=10$，$\beta=0.001$，$G=16$）。

**阶段 3 — 拒绝采样 + SFT**：用第 2 阶段模型生成约 80 万条数据（60 万推理 + 20 万通用），重新 SFT。

**阶段 4 — 多场景 RL**：再次 GRPO，结合规则奖励（数学/代码）+ 神经奖励模型（通用/安全），最终匹配 OpenAI o1-1217。

## 八、直觉速查：关键公式的比喻

**GRPO 优势 $\hat{A}_i = (r_i - \mu) / \sigma$**：想象一次考试，每个学生（回复）的原始分是 $r_i$，但不同科目（prompt）难度不同。z-score 把所有人放到同一尺度——不看绝对分，只看"在这批同学中排第几"。组均值替代了 Critic 的预测，标准差归一化让不同难度的题目产生相当的梯度信号。

**PPO Clip 目标 $\min(r_t \hat{A}_t, \text{clip}(r_t) \hat{A}_t)$**：想象调节旋钮有物理限位器。好的方向最多转到 $1+\varepsilon$，坏的方向最多转到 $1-\varepsilon$，防止一次更新过猛毁掉模型。

**DPO 损失 $-\log \sigma(\beta(\log \frac{\pi_\theta(y_w)}{\pi_{ref}(y_w)} - \log \frac{\pi_\theta(y_l)}{\pi_{ref}(y_l)}))$**：把策略模型想象成打分器——它给好回复的"加分"（相对参考模型的概率增量）应该比给坏回复的更大。DPO 本质是一个二分类器，判据是"谁相对于参考模型被提升得更多"。

**KL 约束 $r(x,y) - \beta \cdot KL(\pi_\theta | \pi_{ref})$**：把 $\pi_{ref}$ 想象成弹簧的自然长度——拉得越远，弹性回复力（KL 惩罚）越大。$\beta$ 是弹簧的刚度系数，防止模型为了刷分跑进离原始分布太远的"荒野"。
