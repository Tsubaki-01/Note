大模型训练是多因素耦合的复杂工程，超参数的选择直接影响收敛速度、最终性能与训练稳定性。本文按优先级系统梳理各类超参数，并提供可直接参考的实践数值。

01

学习率（Learning Rate）

常用值范围

训练阶段典型范围代表模型参考值
预训练（from scratch）1e-4 ~ 3e-4LLaMA-3: 3e-4 / GPT-3: 6e-4
全量微调（Full Fine-tuning）1e-5 ~ 5e-5Alpaca: 2e-5
LoRA / QLoRA1e-4 ~ 3e-4LoRA 论文: 2e-4
RLHF PPO 阶段1e-6 ~ 1e-5InstructGPT: 5e-7

调参逻辑

学习率是最敏感的超参数——过高导致 loss 震荡甚至 NaN，过低则收敛极慢甚至陷入局部最优。

Scaling Law 视角
模型越大，学习率越小。参数量从 1B 到 70B，学习率通常下调 2~5 倍。
Batch Size 联动
Batch 扩大 k 倍 → LR 扩大 √k 倍（大模型推荐），保持训练稳定性。
Warmup 策略
500 ~ 2000 steps
线性升至目标学习率，占总步数 1%~3%。

优化方向

  • 优先使用 Cosine Decay + Warmup 的组合调度策略
  • 训练中途出现 loss spike → 临时降低学习率至 1/3，观察是否恢复
  • 建议使用 LR Finder 或网格搜索确定初始值
  • 分层学习率（Layerwise LR Decay）：底层更小的 LR，逐层递增，微调尤为有效
02

批量大小（Batch Size）

常用值范围

场景Global Batch SizeMicro Batch Size
预训练（A100 集群）1M ~ 4M tokens/step1 ~ 4 per GPU
监督微调（SFT）128 ~ 512 samples1 ~ 8 per GPU
RLHF64 ~ 256取决于显存
ℹ
计算公式Global Batch Size = Micro Batch Size × 梯度累积步数 × GPU 数量

调参逻辑

大 Batch 优势
更稳定的梯度估计，训练曲线平滑；更高的硬件利用率；允许更大的学习率。
大 Batch 代价
泛化能力下降（易陷入 sharp minima）；显存消耗成倍增加。
梯度累积方案
不增加显存的前提下模拟大 Batch，是主流工程选择。

优化方向

  • 预训练以 token 数量衡量 Batch Size（LLaMA 用 4M tokens/step）
  • 微调关注样本多样性，Batch 不宜过大（512~1024 samples 通常够用）
  • 使用 动态批次（Dynamic Batching）按序列长度分桶，提升 GPU 效率
03

序列长度（Context Length）

模型规模预训练长度微调 / 推理长度
7B 级别2K ~ 4K4K ~ 32K
13B ~ 70B4K4K ~ 128K
100B+2K ~ 8K根据需求扩展

序列长度直接影响显存消耗（Attention 计算复杂度为 O(n²)）。长序列扩展通常采用两阶段：先用短序列预训练，再用长序列继续训练。

优化方向

  • 预训练时控制长序列比例（约 5%~10%），避免 GPU 利用率骤降
  • 使用 Flash Attention 2 将显存复杂度降至 O(n)
  • RoPE 通过 YaRN / LongRoPE 低成本扩展上下文
  • 序列长度扩展时，学习率应降至原来的 1/3 ~ 1/5
04

优化器参数（AdamW & Others）

AdamW 核心参数

参数常用值说明
β₁（一阶动量）0.9控制梯度指数移动平均，几乎不需调整
β₂（二阶动量）0.95 ~ 0.999大模型推荐 0.95，提升响应速度
ε（数值稳定性）1e-8 ~ 1e-5BF16 训练建议 1e-6 ~ 1e-5
weight_decay0.01 ~ 0.1通常设为 0.1，Embedding / Bias 不施加
⚠
注意β₂ 过大（0.999）在训练初期会导致梯度估计滞后，对 loss spike 响应变慢。大模型推荐 0.95 ~ 0.98。

新兴优化器对比

优化器特点适用场景
AdamW稳定，成熟通用默认选择
Lion仅符号更新，显存更省超大模型，资源受限
Muon矩阵正交更新实验性，部分场景更快收敛
Sophia二阶优化近似研究阶段

优化方向

  • 大模型（>30B）建议尝试 β₂ = 0.95，提升稳定性
  • 分层 Weight Decay：越靠近输入层越小
  • 8-bit Adam（bitsandbytes）可将优化器状态显存降至 1/4
05

梯度裁剪（Gradient Clipping）

max_grad_norm（通用）
1.0
LLaMA、GPT 系列均使用此值，几乎是大模型标配。
max_grad_norm（不稳定时）
0.3 ~ 0.5
训练频繁出现 loss spike 时临时降低，恢复后可调回 1.0。

梯度裁剪是防止梯度爆炸的核心手段。当梯度全局 L2 范数超过阈值时，按比例缩放所有梯度。

优化方向

  • 监控 grad_norm 是必要的日志指标，突然升高是 loss spike 的预警信号
  • 裁剪值不宜过小，否则会人为压制有效更新，导致收敛变慢
  • 混合精度训练（BF16/FP16）下，裁剪前确保数值缩放正确
06

学习率调度策略（LR Scheduler）

策略适用场景特点
Cosine Decay预训练、大模型通用平滑下降，尾段趋近 0
Linear Decay微调任务简单稳定，可预测
Cosine with Restarts持续训练 / 多任务周期性重新激活
Constant + Warmup快速实验无衰减，适合短期验证
WSD（Warmup-Stable-Decay）最新大模型趋势MiniCPM、Falcon 等采用

WSD 调度策略详解

阶段一
Warmup
线性升至目标 LR
1%~3% 总步数
阶段二（主体）
Stable LR
长期稳定训练
可随时保存 checkpoint
阶段三
Decay
快速收敛冲刺
收益显著
↑ WSD 调度的最大优势：Stable 阶段可灵活延长训练而无需重新设计调度曲线

优化方向

  • 预训练首选 Cosine Decay + Warmup
  • 持续训练（Continual Pre-training）推荐 WSD 策略
  • Warmup 比例控制在总步数的 1% ~ 3%
07

Dropout 与正则化

参数预训练微调
dropout0.0 ~ 0.10.05 ~ 0.1
attention_dropout0.00.0 ~ 0.05
residual_dropout0.0 ~ 0.10.05

大模型预训练阶段通常不使用或使用极小的 Dropout——大规模数据本身提供了充足的正则化效果；超大模型（>30B）几乎完全不使用 Dropout。

优化方向

  • 数据量充足时：dropout = 0.0
  • 数据量少的微调：dropout = 0.05 ~ 0.1
  • Label Smoothing（0.0 ~ 0.1）是比 Dropout 更稳定的正则化替代方案
08

LoRA 专属超参数

参数常用值说明
lora_r（秩）8 ~ 64越大表达能力越强，显存消耗越高
lora_alpha16 ~ 128缩放因子，通常设为 2 × lora_r
lora_dropout0.0 ~ 0.1LoRA 层的 Dropout
应用位置（最小）Q, V仅注意力的查询和值投影
应用位置（最大）Q,K,V,O,FFN接近全量微调效果

alpha / r 比值的意义

# 有效学习率公式
effective_lr = lr × (lora_alpha / lora_r)

# alpha/r = 1：LoRA 贡献与原权重等量级
# alpha/r = 2：放大 LoRA 贡献（最常见设置）
# 推荐：固定 alpha = 2r 作为通用稳健策略

按任务复杂度选择 r

低复杂度任务
r = 8~16
格式遵循、简单问答，资源消耗极低。
高复杂度任务
r = 32~64
代码生成、逻辑推理，需要更强表达能力。
接近全量效果
r = 64~128
应用于所有线性层，预算受限时的最优选择。

优化方向

  • DoRA（Weight-Decomposed LoRA）：进一步提升表达能力
  • RsLoRA：自动将 alpha 缩放为 alpha / √r，更稳定
  • 长上下文任务：扩大应用范围至 FFN 层效果更佳
09

数据相关超参数

参数常用值说明
num_epochs（预训练）1 ~ 3大数据集单轮即可
num_epochs（微调）1 ~ 5小数据集可多轮，防过拟合
data_mix_ratio任务: 通用 = 1~9 : 1防止灾难性遗忘
packing启用将多个短样本打包，提升吞吐
Chinchilla 法则：最优训练 token 数 ≈ 模型参数量 × 20。以 7B 模型为例，最优训练量约为 140B tokens。

优化方向

  • 使用 Data Curriculum（先易后难）可提升最终模型质量
  • 预训练数据去重（MinHash LSH 等）比增加训练轮次更有效
  • 监控训练集 perplexity，过早收敛说明数据多样性不足
  • 微调加入少量通用数据（5%~10%），显著降低灾难性遗忘风险

常见问题诊断

现象可能原因解决方案
Loss NaN / 爆炸学习率过高 / 梯度爆炸降低 LR，增大 grad_clip
Loss 不下降学习率过低 / 数据质量差升高 LR，检查数据
Loss 下降后反弹LR 未衰减 / 数据噪声启用 LR Decay，清洗数据
过拟合数据量不足 / 轮数过多增大正则化，减少 epochs
GPU 利用率低Batch 过小 / 序列不均匀增大 Batch，启用 packing

工程经验总结

  • 先用 1B 模型验证超参配置，再扩展到 7B / 13B
  • 监控关键指标：loss、grad_norm、lr、tokens/sec、GPU 利用率
  • 至少保存最近 3 个 checkpoint，防止 loss spike 后无法回退
  • 使用确定性种子（seed = 42），对比实验必须固定
  • 批量大小、序列长度不要一步到位，渐进式扩展
  • 理解每个超参数背后的数学直觉，比死记推荐值更重要
