# Python 工程化最佳实践：模块封装与单例导出

## 1. 核心理念

在 Python 中开发工具类模块（如 `PromptManager`, `ConfigLoader`, `DBClient`）时，不应让外部调用者感知内部的文件结构。应通过 `__init__.py` 暴露接口，实现**“高内聚，低耦合”**。

## 2. 目录结构范式

推荐将功能封装为独立的子包（Sub-package），通过 `__init__.py` 控制对外暴露的内容。

```
my_project/
└── src/
    └── my_module/           # 功能子包 (例如: prompts, database)
        ├── __init__.py      # [关键] 负责对外暴露接口
        ├── core.py          # [核心] 类的具体实现 (文件名随意，外部不感知)
        └── utils.py         # 辅助函数 (仅供内部使用，不对外暴露)
```

## 3. 代码实现模式

### 步骤 A：实现类并预实例化 (Internal)

在内部文件（如 `core.py`）中完成逻辑实现，并创建一个**默认实例**。

```py
# src/my_module/core.py

class ServiceManager:
    def __init__(self):
        self._cache = {}

    def run(self, data):
        print(f"Processing {data}...")

# === 单例模式核心 ===
# 在模块层级直接实例化，保证全局使用同一个对象（共享内存/缓存）
default_manager = ServiceManager()
```

### 步骤 B：通过 Init 暴露 (Interface)

在 `__init__.py` 中导入该实例，建立对外接口。

```Python
# src/my_module/__init__.py

# 1. 相对导入：从当前包的文件中导入实例
from .core import default_manager

# 2. (可选) 如果允许外部创建新实例，也可以导出类
from .core import ServiceManager

# 3. 明确导出列表：控制 import * 的行为，隐藏不相关的 utils
__all__ = ["default_manager", "ServiceManager"]
```

### 步骤 C：外部调用 (Usage)

外部代码调用时，路径短且清晰，无需知道 `core.py` 的存在。

```Python
# main.py

# ✅ 推荐：直接从包导入实例
from my_module import default_manager

default_manager.run("test")

# ❌ 避免：不要去引用内部文件名（一旦重构文件名，这就报错了）
# from my_module.core import default_manager 
```

------

## 4. 为什么要这么做？（核心优势）

| **优势维度** | **说明**                                                     |
| ------------ | ------------------------------------------------------------ |
| **简化引用** | 调用者只需 `from module import instance`，无需记忆复杂的内部文件名（如 `manager.py` 或 `impl.py`）。 |
| **重构安全** | 如果未来觉得 `core.py` 太大，拆分为 `core_v1.py` 和 `core_v2.py`，只需修改 `__init__.py` 的指向，**外部调用代码一行都不用改**。 |
| **单例共享** | 确保项目中所有地方引用的 `default_manager` 都是同一个实例。这对**配置管理**、**数据库连接池**、**缓存（Cache）**至关重要。 |
| **API 清洁** | 隐藏了内部的辅助文件（如 `utils.py`、`constants.py`），保持对外接口的整洁。 |

## 5. 适用场景

- **Prompt 管理**：全局只需加载一次模板文件，避免重复 IO。
- **配置加载 (Config)**：全局只需读取一次 `.env` 或 `yaml`，并在内存中共享配置。
- **日志 (Logger)**：统一配置好格式的 Logger 实例。
- **数据库/API 客户端**：维持长连接的 Client 实例。
