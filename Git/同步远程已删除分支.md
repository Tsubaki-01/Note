当远程分支被删除后，本地会残留该分支的**远程跟踪引用**（比如 `origin/old-feature`），需要同步清理这些无效引用，让本地分支列表和远程保持一致。下面给出分步骤的实操命令，新手也能直接执行：

### 一、核心同步操作（推荐一键清理）
这是最常用、最高效的方式，会自动拉取远程最新状态并清理无效分支引用：
```bash
# 拉取远程最新分支信息，并自动删除本地缓存的、远程已删除的分支引用
# --prune（简写 -p）是核心参数，意为“修剪”无效引用
git fetch --prune  # 简写：git fetch -p
```

执行后，你可以通过以下命令验证效果：
```bash
# 查看本地所有远程跟踪分支（origin/xxx 形式）
git branch -r
# 查看本地所有分支（含本地分支+远程跟踪分支）
git branch -a
```
此时已删除的远程分支（如 `origin/old-feature`）会从列表中消失。

### 二、手动清理单个远程分支引用（按需使用）
如果只想清理某一个特定的、远程已删除的分支引用，而非全部：
```bash
# -d：删除；-r：针对远程跟踪分支
git branch -dr origin/要清理的分支名
# 示例：清理远程已删除的 old-feature 分支引用
git branch -dr origin/old-feature
```

### 三、额外：清理本地对应的分支（可选）
如果本地有基于已删除远程分支创建的**本地分支**（比如 `old-feature`），且该分支已合并到主分支、无保留价值，可手动删除：
```bash
# 安全删除（仅删除已合并到当前分支的本地分支）
git branch -d 本地分支名
# 强制删除（即使未合并，谨慎使用）
git branch -D 本地分支名
```

### 四、进阶：设置默认自动清理（可选）
如果想让 `git fetch` 每次都自动执行 prune（无需手动加 `-p`），可以配置 Git 全局参数：
```bash
git config --global fetch.prune true
```
配置后，每次执行 `git fetch` 都会自动同步远程分支状态，清理无效引用。

### 总结
1. 核心同步命令是 `git fetch -p`（`--prune`），一键清理本地缓存的远程已删除分支引用；
2. 手动清理单个远程分支引用用 `git branch -dr origin/分支名`；
3. 本地对应分支可通过 `git branch -d/-D` 删除，`-d` 安全、`-D` 强制。
